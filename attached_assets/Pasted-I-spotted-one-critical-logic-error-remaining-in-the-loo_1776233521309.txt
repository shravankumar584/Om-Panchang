The "Skeleton Check" Bug
In your current fill function, you have this line:

JavaScript

Why this is a problem: When you click "Next Month," you call setCalendarDays([]) inside the Maps function. If the async fill starts and its first batch finishes before React has finished rendering the new skeleton, prev will be an empty array []. prev[idx] will be undefined, the code will return prev, and that specific day's data is permanently lost for that render cycle.
